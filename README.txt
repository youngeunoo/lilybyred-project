01/08
시작할 때 보이는 main부분에서 스크롤을 내릴 때 fullpage처럼 내려가도록 작업함.
처음에 그냥 .from으로 작업했다가, 스크롤을 따라 여러 번에 나눠 opacity와 transform이 적용되길래, onLeave 함수를 이용함.

section02 영역이 나타날 때 서서히 나타날 수 있도록 onEnter 함수를 이용해 opacity 값이 0에서 1로 변화되도록 함

section03-05 부분은 pinning 후 각 아이템들에 opacity 값과 scale 값을 줘서 나타나도록 함. 

magazine 영역에 도달해서는 스크롤을 좀 더 내린 후 요소들이 보일 수 있게 start값을 조절함.
해당 영역에서 아이템의 움직임이 보일 수 있게 timeline과 pin을 사용함

process 영역은 세 영역이 이어지는 것처럼 보일 수 있도록 숫자 박스에 선을 이었고, 스크롤에 따라 글과 사진이 보일 수 있도록 함

warning과 outro에서는 배경색이 자연스럽게 바뀌어야 하는데, 각 영역에 백그라운드컬러를 주니까 연결 부분이 보일 수밖에 없었음. 찾아보니 영역을 벗어날 때 body의 백그라운드컬러를 바꾸는 방법이 적절할 것 같아 적용해봤더니 자연스러웠음.
다른 영역도 onEnter, onLeave 함수를 적절히 사용하여 body 백그라운드컬러를 변경해 각 영역의 이동이 자연스럽도록 하였음.

warning 부분 요소들이 처음엔 안 보였다가 해당 영역에 도달하면 보이도록 수정함.

01/10
main 영역과 section01 영역을 합쳐서 메인에서 넘어갈 때 스크롤이 내려가는 게 눈에 보이는 문제를 해결함.
magazine 부분에 요소가 너무 빨리 움직이는 걸 고치고 싶어서 duration, scrub 값도 바꿔보고 스크롤 위치도 바꿔봤는데 그대로였음.
process 영역의 사진 구조와 배치가 애매해서 새로 구상함. 사진을 폴더처럼 디자인하고 아이폰 아이콘을 활용했는데 괜찮은 건지 모르겠음.
outro 구조도 더 보기 좋게 정렬을 다시 함.
warning 영역 포인트로 warning 패널이 스크롤에 맞춰 깜빡거리게 했는데 이것도 조금 더 자연스럽게 바꿔야 할 것 같음. 그리고 영역 벗어날 때 너무 부자연스럽게 바디 컬러가 바뀜.
